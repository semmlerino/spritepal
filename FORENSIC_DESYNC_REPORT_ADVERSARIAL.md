**Required Invariants**
- Sprite identity must be composite (offset + source_type + capture frame/time + palette ID + tile ID) and preserved across selection → preview → editor → canvas.
- FILE offset from capture must remain immutable; ROM offset and alignment corrections must be stored separately, not overwriting capture identity.
- “Current sprite” operations must use editor pixel state (`image_model`) when in edit mode, not stale ROM bytes.
- Asset browser activation must preserve source_type and category; reselection must not silently flip category when offsets collide.
- Thumbnail/cache keys must not collapse across categories unless explicitly intended and proven safe.
- Palette/tile IDs shown to users must be provenance-tracked; fallback palettes must be labeled so correctness isn’t inferred.
