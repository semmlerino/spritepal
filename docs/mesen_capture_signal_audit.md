- `LogWatcher.offset_discovered` — emitted in `_read_new_content` and `_check_offset_file` after `_recent_captures`/`_seen_offsets` update; data: `CapturedOffset` (FILE offset, frame, timestamp, raw_line, rom_checksum); expected: capture UI per `Mesen2Module.connect_signals` docstring; actual receivers: `MesenCapturesSection.add_capture` via `Mesen2Module.connect_signals` `ui/rom_extraction/widgets/mesen_captures_section.py`, `ROMWorkflowController._on_offset_discovered` `ui/sprite_editor/controllers/rom_workflow_controller.py`, relay to `Mesen2Module.offset_discovered` `ui/rom_extraction/modules/mesen2_module.py` (no prod listeners); timing: poll/file change, `last_offset.txt` checked before log file; assumptions: offset is FILE-based `core/mesen_integration/log_watcher.py`.
- `LogWatcher.watch_started` — emitted at end of `start_watching` after polling timer starts; data: none; expected: update “watching” UI; actual receivers: relay to `Mesen2Module.watch_started` `ui/rom_extraction/modules/mesen2_module.py` (unused), plus `widget.set_watching(True)` via lambda in `Mesen2Module.connect_signals` `ui/rom_extraction/modules/mesen2_module.py` → `MesenCapturesSection.set_watching` `ui/rom_extraction/widgets/mesen_captures_section.py` → `ROMExtractionPanel.mesen2_watching_changed` `ui/rom_extraction_panel.py` → `StatusBarManager.set_mesen2_watching` `ui/managers/status_bar_manager.py`; timing: fires even if directory missing; assumptions: watcher started once `core/mesen_integration/log_watcher.py`.
- `LogWatcher.watch_stopped` — emitted in `stop_watching` when `_log_path` cleared; data: none; expected: update “watching” UI; actual receivers: same relay/chain as `watch_started` (relay unused, widget update via lambda) `core/mesen_integration/log_watcher.py`.

**Lifecycle / Ordering (Key Points)**
- Startup: `MainWindow` creates `LogWatcher`, `Mesen2Module`, `ROMExtractionPanel`, and `ROMWorkflowController`; `ROMWorkflowController._connect_log_watcher` starts watching early, so log watcher can emit before capture UI is wired `ui/main_window.py` `ui/sprite_editor/controllers/rom_workflow_controller.py` `ui/rom_extraction/modules/mesen2_module.py`.
- Capture discovery: `LogWatcher` emits `offset_discovered` → capture list (`RecentCapturesWidget`) and asset browser (`ROMWorkflowController`), independently `core/mesen_integration/log_watcher.py` `ui/rom_extraction/widgets/mesen_captures_section.py` `ui/sprite_editor/controllers/rom_workflow_controller.py`.
- ROM load: ROM extraction panel sets `_smc_offset` and starts capture thumbnail worker, then calls `request_all_thumbnails` `ui/rom_extraction_panel.py`; sprite editor `load_rom` starts its thumbnail worker and re-syncs captures to the asset browser `ui/sprite_editor/controllers/rom_workflow_controller.py`.
- Selection: capture list double-click → `open_in_sprite_editor` → `jump_to_offset` → asset browser selection → preview request → preview_ready → optional `open_in_editor` `ui/rom_extraction_panel.py` `ui/main_window.py` `ui/workspaces/sprite_editor_workspace.py` `ui/sprite_editor/controllers/rom_workflow_controller.py`.
- Teardown: ROM extraction panel cleanup stops thumbnail worker; ROM workflow cleanup stops preview/thumbnail workers; log watcher remains active unless externally stopped `ui/rom_extraction_panel.py` `ui/sprite_editor/controllers/rom_workflow_controller.py`.

**Flags**
- **Emitted but never received (production)**: `LogWatcher.directory_warning` `core/mesen_integration/log_watcher.py`; `Mesen2Module.offset_discovered/watch_started/watch_stopped/error_occurred` `ui/rom_extraction/modules/mesen2_module.py`; `MesenCapturesSection.save_to_library_requested` and `RecentCapturesWidget.save_to_library_requested` `ui/rom_extraction/widgets/mesen_captures_section.py` `ui/components/panels/recent_captures_widget.py`; `ROMWorkflowController.workflow_state_changed` and `ROMWorkflowController.sprite_extracted` `ui/sprite_editor/controllers/rom_workflow_controller.py`; `ThumbnailWorkerController.thumbnail_ready` in this flow (worker signal is used directly) `ui/workers/batch_thumbnail_worker.py`.
- **Expected but never emitted**: none explicitly surfaced in this path; several signals only emit on error conditions (`LogWatcher.error_occurred`, `SmartPreviewCoordinator.preview_error`) and may be silent in normal runs.
- **Stale/invalid data paths**
  - `RecentCapturesWidget` emits normalized ROM offsets, but `MainWindow._on_open_in_sprite_editor` treats `open_in_sprite_editor`’s offset as FILE offset and normalizes again; this double-normalizes when SMC headers exist `ui/components/panels/recent_captures_widget.py` `ui/rom_extraction_panel.py` `ui/main_window.py` `ui/sprite_editor/controllers/rom_workflow_controller.py`.
  - `ROMWorkflowController.capture_offset_adjusted` emits ROM offsets, but `RecentCapturesWidget.update_capture_offset` searches by FILE offset; alignment adjustments likely fail to update the capture list `ui/sprite_editor/controllers/rom_workflow_controller.py` `ui/components/panels/recent_captures_widget.py` `ui/rom_extraction/widgets/mesen_captures_section.py` `ui/main_window.py`.
  - `LogWatcher.get_capture_by_offset` compares against FILE offsets but its docstring claims ROM offset; in `MainWindow._on_open_in_sprite_editor`, capture lookup is performed with a (normalized) ROM offset, so frame-based display names will often be missing `core/mesen_integration/log_watcher.py` `ui/main_window.py`.
  - Captures added before ROM load are normalized with `_smc_offset=0`; later `set_smc_offset` does not re-normalize existing items, so `thumbnail_requested` and selection offsets can remain wrong for SMC-headered ROMs `ui/components/panels/recent_captures_widget.py` `ui/rom_extraction_panel.py`.
- **Fragile thumbnail chains**
  - Capture thumbnails are requested on `thumbnail_requested`, but `ROMExtractionPanel._on_thumbnail_requested` is a no-op until `_thumbnail_controller` exists; it relies on a later `request_all_thumbnails` to recover `ui/rom_extraction_panel.py` `ui/components/panels/recent_captures_widget.py`.
  - Asset browser thumbnails only queue if `_thumbnail_controller` exists; captures added before `load_rom` won’t queue until `load_rom` clears and re-syncs the browser `ui/sprite_editor/controllers/rom_workflow_controller.py`.
- **UI state not reflecting data**
  - Mesen capture list offsets may not update after alignment correction (signal data mismatch), so the list can stay on the original FILE offset while the editor operates on the aligned ROM offset `ui/components/panels/recent_captures_widget.py` `ui/sprite_editor/controllers/rom_workflow_controller.py`.
  - Capture list display text shows FILE offsets while selection emits ROM offsets; main window assumes FILE offsets, so the editor can open a different sprite than the one displayed for SMC-headered ROMs `ui/components/panels/recent_captures_widget.py` `ui/main_window.py`.

**Optional: High-Level Signal Architecture Suggestions**
- Standardize offset units in signal names/docs (`file_offset` vs `rom_offset`) and/or introduce a single “normalized offset” adapter to avoid double-normalization and mismatched receivers.
- Consolidate capture-to-editor bridging into one relay (either `Mesen2Module` or `ROMWorkflowController`) to reduce duplicate watchers and ambiguities about which signal path is authoritative.
- Expose a single thumbnail request/response interface (controller-level) so UI doesn’t depend on direct worker signal wiring.
